# Форматы данных

## Входной файл

В строке задаются исходная и конечная вершины

Разделителем может быть запятая, пробел или табуляция

Допускаются пустые строки, комментарии с `#`, окончания строк CRLF и один заголовок перед первым ребром

```csv
from,to
-10,20
20,30
```

Подойдёт и обычный список рёбер SNAP

```text
# FromNodeId ToNodeId
-10 20
20 30
```

Идентификаторы должны помещаться в знаковый `int32`

Повторные рёбра удаляются, петли сохраняются

Исходный файл читается два раза и не должен меняться между проходами

## Подготовленный граф

После команды `preprocess` появляется каталог с двоичным представлением

- `manifest.json` — описание графа и диапазонов частей
- `vertices.bin` — исходные идентификаторы вершин
- `outdegrees.bin` — исходящие степени
- `shard-NNNNN.bin` — рёбра одного диапазона конечных вершин

Подробное устройство файлов описано в [спецификации внутреннего формата](file-format.md)

Каталог сначала строится под временным именем и появляется по заданному пути только после успешной проверки

## Результат

PageRank записывается в CSV

```csv
vertex,rank
-10,0.125
20,0.375
30,0.5
```

Строки идут по возрастанию исходных идентификаторов

Ранг записывается с точностью, достаточной для восстановления значения `double`

Существующий файл программа не перезаписывает
