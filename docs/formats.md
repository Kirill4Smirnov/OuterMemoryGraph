# Форматы данных

## Входной список рёбер

В каждой строке задаются идентификаторы исходной и конечной вершины

Разделителем служит запятая, пробел или табуляция

Допускаются пустые строки, комментарии с `#`, окончания строк CRLF и один заголовок перед первым ребром

```csv
from,to
-10,20
20,30
```

Идентификатор имеет тип `int32`

Повторные рёбра удаляются, а петли сохраняются

## Подготовленный граф

Каталог содержит следующие файлы

- `manifest.json` — описание графа и диапазонов частей
- `vertices.bin` — исходные идентификаторы в порядке внутренних номеров
- `outdegrees.bin` — исходящие степени типа `uint32`
- `shard-NNNNN.bin` — рёбра одного диапазона конечных вершин

Одно ребро занимает восемь байт

```text
uint32 destination
uint32 source
```

Поля `destination` и `source` хранят внутренние номера конечной и исходной вершины

Рёбра отсортированы сначала по конечной, затем по исходной вершине

Диапазоны частей не пересекаются и вместе покрывают `[0, V)`

Первая версия формата рассчитана на Linux x64 с порядком байтов от младшего к старшему

Перед вычислением PageRank размеры файлов сверяются с `manifest.json`, а во время чтения проверяются порядок и диапазоны рёбер

Каталог строится под временным именем и публикуется только после успешной проверки

## Результат

PageRank записывается в CSV

```csv
vertex,rank
-10,0.125
20,0.375
30,0.5
```

Строки отсортированы по исходному идентификатору

Ранг записывается с точностью, достаточной для восстановления значения `double`

Сначала создаётся временный файл, затем он переименовывается в заданный путь
