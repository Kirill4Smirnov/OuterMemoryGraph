# Алгоритм и оценки

## Почему выбран PageRank

PageRank оценивает важность вершины по входящим связям и применяется к ориентированным графам

PageRank здесь удобен

- значение вычисляется для каждой вершины
- одна итерация требует одного прохода по рёбрам
- результат занимает `O(V)` памяти
- существуют эталонные результаты LDBC Graphalytics

## Формула

Для вершины `v` используется выражение

$$
r_{t+1}(v) =
\frac{1-d}{N}
+ d \sum_{u \to v}\frac{r_t(u)}{\deg^+(u)}
+ \frac{d}{N}\sum_{w:\deg^+(w)=0}r_t(w)
$$

Здесь `N` обозначает число вершин, `d` — коэффициент затухания, а `deg+(u)` — исходящую степень вершины

Начальный ранг каждой вершины равен `1 / N`

Ранг вершин без исходящих рёбер равномерно распределяется между всеми вершинами, поэтому общая сумма остаётся равна единице

## Одна итерация

- вычисляется сумма рангов вершин без исходящих рёбер
- для каждой вершины задаётся базовый вклад
- файлы графа распределяются между рабочими потоками
- каждый поток последовательно читает свой файл и собирает входящие вклады
- L1-разность и сумма рангов сводятся в порядке номеров файлов

Все входящие рёбра одной вершины лежат подряд

Их вклад складывается с компенсацией погрешности в типе повышенной точности `long double`

## Условие остановки

Можно задать точное число итераций через `--iterations`

Второй вариант — остановка при достижении заданной L1-разности соседних векторов

$$
\lVert r_t-r_{t-1}\rVert_1
$$

Эта величина не равна точной ошибке относительно стационарного решения

Для коэффициента затухания `d` подходит консервативная оценка

$$
\lVert r_t-r^*\rVert_1
\leq
\frac{\lVert r_t-r_{t-1}\rVert_1}{1-d}
$$

При `d = 0.85` и L1-разности `1e-8` верхняя граница равна примерно `6.67e-8`

## Сложность

Пусть `V` — число вершин, `E` — число уникальных рёбер, `K` — число итераций

- предварительная обработка требует до `O(E log E)` сравнений
- одна итерация PageRank занимает `O(V + E)` операций
- весь расчёт занимает `O(K(V + E))` операций
- за одну итерацию с диска читается `O(E)` данных
- в оперативной памяти находится около `20 * V` байт плюс небольшие буферы

Каждое ребро читается ровно один раз за итерацию, поэтому оценка одной итерации не меняется для графов разной плотности

Число итераций зависит от устройства конкретного графа

## Ограничения

- список рёбер может быть больше оперативной памяти, но массивы состояния должны в неё помещаться
- из списка рёбер нельзя восстановить изолированные вершины
- один узел с очень большой входящей степенью обрабатывается одним потоком
- промежуточные ранги не сохраняются для продолжения после сбоя
- внутренний формат зависит от порядка байтов платформы
