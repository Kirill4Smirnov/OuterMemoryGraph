# Проверка корректности и памяти

## Автоматические тесты

В проекте есть 12 проверок

- чтение CSV и SNAP
- отрицательные `int32` идентификаторы и ошибочные строки
- контрольная сумма входного списка рёбер
- перенумерация вершин, удаление повторов и обработка петель
- одинаковый результат сортировки при одном и нескольких потоках
- перераспределение ранга вершин без исходящих рёбер
- одинаковый PageRank при разном числе потоков
- вершина с 50 000 входящих и 50 000 исходящих рёбер
- эталонный граф LDBC Graphalytics
- проверка описания подготовленного графа и рабочих каталогов

Для сборки этих тестов нужен Catch2 версии 3

```bash
cmake -S . -B build-test \
  -G Ninja \
  -DCMAKE_BUILD_TYPE=Debug \
  -DOMG_BUILD_TESTS=ON
cmake --build build-test
ctest --test-dir build-test --output-on-failure
ctest --test-dir build-test --output-on-failure --parallel 12
```

В стандартных пакетах Ubuntu 22.04 находится более старая версия Catch2

Для обычной сборки программы Catch2 не нужен, а проверку по Graphalytics можно запустить по инструкции из README

## Сравнение с Graphalytics

Встроенный тестовый граф содержит 50 вершин и 246 рёбер

Эталон рассчитан для коэффициента затухания `0.85` и 14 итераций

Допустимая относительная ошибка равна `1e-4`

Результат текущей версии

- максимальная относительная ошибка `1.2677113296e-06`
- максимальная абсолютная ошибка `2.7235540538e-08`
- сумма рангов `1.0`

Полученная относительная ошибка примерно в 79 раз меньше допустимой

## Многопоточность

Один тест строит граф с одним и четырьмя потоками сортировки

Двоичные файлы и описание графа должны полностью совпасть

Другой тест запускает PageRank с одним и четырьмя потоками и сравнивает результаты побайтово

## Ограничение памяти

Программа заранее оценивает собственные выделения по значению `--memory-mb`

Жёсткое ограничение задаётся контрольной группой Linux

```bash
scripts/check_memory.sh \
  ./build/omg \
  graphs/livejournal.omg \
  /tmp/livejournal-memory-check.csv \
  16 \
  2
```

Проверенный запуск на `soc-LiveJournal1`

- вершин `4 847 571`
- рёбер `68 993 773`
- итераций `2`
- наибольший расход памяти `127.9 МиБ`
- файл подкачки не использовался
- сумма рангов `1.0`

Граф содержит 68 993 773 ребра, а процесс завершился при ограничении 128 МиБ без файла подкачки

Отдельный полный запуск на Graphalytics прошёл в контейнере Ubuntu 22.04 с тем же ограничением

## Дополнительные проверки

- отладочная и оптимизированная сборки
- AddressSanitizer и UndefinedBehaviorSanitizer
- параллельный запуск CTest
- `clang-format`, `clang-tidy`, Ruff и ShellCheck

Все 12 тестов проходят в обычной сборке и под санитайзерами

## Проверка файла с результатом

Без эталона скрипт проверяет порядок идентификаторов, конечность рангов, отсутствие отрицательных значений и сумму

```bash
python3 scripts/validate_output.py pagerank.csv
```

При наличии эталона скрипт также считает абсолютную и относительную ошибку

```bash
python3 scripts/validate_output.py \
  pagerank.csv \
  --expected reference.csv \
  --relative-tolerance 1e-4
```
