# Проверка результата

## Автоматические тесты

В проекте есть 12 тестов

Они проверяют

- чтение CSV и списков рёбер SNAP
- отрицательные `int32` идентификаторы и ошибочные строки
- обнаружение изменения входного файла между проходами
- перенумерацию вершин, удаление повторов и сохранение петель
- одинаковый результат сортировки при разном числе потоков
- перераспределение ранга вершин без исходящих рёбер
- одинаковый PageRank при разном числе потоков
- вершину с 50 000 входящих и 50 000 исходящих рёбер
- эталонный граф LDBC Graphalytics
- описание подготовленного графа и проверку рабочих каталогов

Для тестовой сборки нужен Catch2 версии 3

```bash
cmake -S . -B build-test \
  -G Ninja \
  -DCMAKE_BUILD_TYPE=Debug \
  -DOMG_BUILD_TESTS=ON
cmake --build build-test
ctest --test-dir build-test --output-on-failure
ctest --test-dir build-test --output-on-failure --parallel 12
```

В Ubuntu 22.04 из обычного репозитория устанавливается более старая версия Catch2

Для проверки самой программы Catch2 не нужен — эталонный пример можно запустить по инструкции из README

## Эталон LDBC Graphalytics

Проверочный граф содержит 50 вершин и 246 рёбер

Эталон рассчитан для коэффициента затухания `0.85` и 14 итераций

Допустимая относительная ошибка равна `1e-4`

Результат проекта

- максимальная относительная ошибка `1.2677113296e-06`
- максимальная абсолютная ошибка `2.7235540538e-08`
- сумма рангов `1.0`

## Воспроизводимость

Один тест готовит граф с одним и четырьмя потоками и сравнивает двоичные файлы

Другой запускает PageRank с одним и четырьмя потоками и сравнивает итоговые CSV побайтово

Порядок сложения вкладов внутри вершины фиксирован, поэтому расписание потоков не меняет результат

## Ограничение памяти

Параметр `--memory-mb` проверяет крупные выделения самой программы

Настоящий предел памяти задаётся контрольной группой Linux

```bash
scripts/check_memory.sh \
  ./build/omg \
  graphs/livejournal.omg \
  /tmp/livejournal-memory-check.csv \
  16 \
  2
```

Проверенный запуск на `soc-LiveJournal1`

- `4 847 571` вершин
- `68 993 773` ребра
- 2 итерации
- наибольший расход памяти `127.9 МиБ`
- файл подкачки не использовался
- сумма рангов `1.0`

Такой же запуск малого эталонного графа проверен в контейнере Ubuntu 22.04

## Проверка CSV

Без эталона скрипт проверяет порядок идентификаторов, конечность и неотрицательность рангов, а также их сумму

```bash
python3 scripts/validate_output.py pagerank.csv
```

С эталоном дополнительно считаются абсолютная и относительная ошибки

```bash
python3 scripts/validate_output.py \
  pagerank.csv \
  --expected reference.csv \
  --relative-tolerance 1e-4
```

Код также проверялся в отладочной и оптимизированной сборках, с AddressSanitizer и UndefinedBehaviorSanitizer
